CourseWave: LMS Platform — System Design Document
==================================================
Last updated: 2026-04-04

1. PRODUCT OVERVIEW
-------------------
CourseWave is an all-in-one learning platform that combines four things competitors sell separately:
  1. Course marketplace (like Udemy)
  2. Community chat (like Discord)
  3. Live mentoring sessions (like Calendly + video)
  4. Project-based portfolio building (like Buildspace)

Additional differentiators:
  - AI-powered learning roadmap generation
  - Article/blog publishing platform (like Medium for devs)
  - Completion certificates with verification

Target: Developers and tech learners.
Business model: Hybrid — individual course purchases + monthly subscriptions.
Operator: Solo developer (Aman), launching as SaaS for side income.

3. TECH STACK (ACTUAL)
----------------------

Frontend:
  - Framework: Next.js 15.3.8 (App Router, Turbopack)
  - UI: React 18.3.1, Tailwind CSS v4.1, Framer Motion 12
  - Components: shadcn/ui (Radix), Tremor (charts), Lucide (icons)
  - State: Zustand with SSR-safe localStorage persistence
  - Data fetching: TanStack React Query v5
  - Auth: NextAuth v4 (Google + GitHub OAuth), JWT tokens synced to Zustand
  - Real-time: Socket.io-client
  - Rich text: EditorJS (articles), DOMPurify (sanitization)
  - Fonts: Poppins (body), DM Sans (headings)
  - Hosting: Vercel

Backend:
  - Runtime: Node.js 22 (LTS)
  - Framework: Express.js with TypeScript
  - ORM: Prisma with PostgreSQL
  - Auth: JWT (access + refresh tokens), bcrypt password hashing
  - OAuth: Backend endpoint POST /api/auth/oauth (receives NextAuth callback)
  - Real-time: Socket.io (community chat)
  - Payments: Stripe Checkout + Webhooks + Subscriptions
  - Email: SendGrid
  - File storage: Supabase Storage (videos, images, documents)
  - Validation: Zod (request schemas + env config)
  - Hosting: Koyeb

NOT using (decided against):
  - NestJS (Express is simpler for solo dev)
  - Supabase Auth (built custom JWT system for full control)
  - Firebase (Socket.io gives better control for chat)
  - AWS S3 (Supabase Storage is simpler)
  - Redis (commented out — may add later for caching/presence)


4. DATABASE SCHEMA (KEY MODELS)
-------------------------------

User System:
  User → UserRole (USER | INSTRUCTOR | ADMIN)
  User → OAuthAccount (Google, GitHub — multiple providers per user)
  User → Token (access, refresh, verify_email, reset_password)
  User.password is nullable (OAuth users don't need passwords)

Course System:
  Course → CourseSection → Chapter (video, text, quiz, audio)
  Course → Category (single)
  Course → Review (rating 1-5, auto-aggregates to Course.averageRating)
  Course → Enrollment (PENDING → ACTIVE → COMPLETED)
  Course → CourseProgress → ChapterProgress (per-user tracking)
  Course → Instructor (userId, bio, expertise, earnings)

Payment System:
  User → StripeCustomer (one-to-one)
  User → Cart → CartItem (course references)
  User → Wishlist → WishlistItem
  User → CoursePurchase (one-time buys)
  User → UserSubscription → SubscriptionPlan → PlanCourse (tier-gated access)
  Instructor → InstructorEarning (per-course, per-period)

Community System:
  Community → CommunityMember (ADMIN | MODERATOR | MEMBER)
  Community → Message (content, attachments JSON, parentId for threads)
  Message → Reaction (LIKE | DISLIKE | LOVE | LAUGH, toggle per user)
  Community → PinnedMessage
  Community → ModerationLog

Article System:
  Blog → BlogComment → CommentLike
  Blog → BlogLike, BlogView, BlogRating, BlogReport
  User → SavedArticle
  User → UserFollower (follow/unfollow authors)

Project System:
  Project → ProjectSubmission → ProjectFeedback
  Project belongs to Course + Instructor
  Fields: milestones, rubric, difficulty, technologies, prerequisites

Session System:
  Session → SessionBooking (with Stripe payment)
  Session has rtcType (WEBRTC | STREAM | ZEGO | AGORA | CUSTOM)
  Fields: rtcRoomId, rtcToken, rtcConfig

Notification System:
  Notification (userId, role, type, title, message, link, channel, read status)
  Triggered by: enrollment, completion, review, payment
  Channels: in-app (implemented), email (planned)


5. AUTHENTICATION FLOW
----------------------

Email/Password:
  1. POST /api/auth/register → hash password → create User + UserRole → generate JWT pair
  2. POST /api/auth/login → verify password → generate JWT pair
  3. Access token in Authorization header, refresh via POST /api/auth/refresh
  4. Token stored in Zustand userStore (persisted to localStorage)

OAuth (Google/GitHub):
  1. Frontend: NextAuth handles OAuth redirect + callback
  2. NextAuth signIn callback → POST /api/auth/oauth with provider profile
  3. Backend: create User if new, link OAuthAccount, generate JWT pair
  4. JWT tokens attached to NextAuth session → synced to Zustand via useOAuthSync hook
  5. OAuth users have password: null — cannot use email/password login

Security measures:
  - Rate limiting: 5 attempts/15min on login/forgot/reset, 10/15min on register
  - ADMIN role blocked from registration schema
  - verifyToken middleware on all protected routes
  - CORS restricted in production (HTTP + Socket.io)
  - XSS sanitized with DOMPurify for user-generated HTML content
  - No hardcoded secrets (all from Zod-validated env)


6. PAYMENT SYSTEM
-----------------

Model: Hybrid (individual purchases + subscriptions)

Individual Course Purchase:
  1. User clicks "Enroll Now" on course page
  2. If course.isFree → direct enrollment (ACTIVE status)
  3. If paid → POST /api/courses/:id/checkout → create Stripe Checkout Session
  4. User redirected to Stripe → pays → Stripe webhook fires
  5. Webhook: checkout.session.completed → activateEnrollmentAfterPayment()
  6. Enrollment: PENDING → ACTIVE, notifications sent to student + instructor

Subscription Plans (3 tiers):
  Free:     $0    — Browse, first chapter free, 3 AI roadmaps/month, read articles
  Learner:  $12/mo ($99/yr) — All standard courses, certificates, community, projects
  Pro:      $29/mo ($249/yr) — All courses, priority feedback, 4 live sessions/month

Instructor Plans:
  Free:     2 courses max, 70% revenue share
  Premium:  $19/mo ($149/yr) — Unlimited courses, analytics, 85% revenue share

Revenue target at 6 months: $2,000-2,500/month


7. REAL-TIME SYSTEM (COMMUNITY CHAT)
-------------------------------------

Transport: Socket.io over WebSocket (with polling fallback)
Auth: JWT token in socket handshake, verified by verifySocketToken middleware
Rooms: community:{communityId}

Events (Client → Server):
  joinCommunity   → Join room, receive last 50 messages, set online
  sendMessage     → Create message (with optional parentId for replies, attachments JSON)
  editMessage     → Edit own message (sender verification)
  deleteMessage   → Delete own message or admin/moderator can delete any
  reactToMessage  → Toggle reaction (LIKE/DISLIKE/LOVE/LAUGH)
  togglePinMessage → Pin/unpin (any member)
  typing          → Broadcast typing indicator to room

Events (Server → Client):
  messages:init     → Initial message history (last 50)
  message:new       → New message broadcast
  message:edited    → Edited message broadcast
  message:deleted   → Deleted message ID broadcast
  message:reaction  → Full reaction state for a message
  message:pinned    → Pin/unpin status change
  typing:update     → Typing indicator (userId, name, isTyping)
  presence:update   → Online/offline status
  error:message     → Error feedback to sender

Disconnect: Sets CommunityMember.isOnline = false for all joined rooms, broadcasts presence.


8. KEY USER FLOWS
-----------------

Student Journey:
  Landing → Browse/Search → Course Detail → Enroll (free/paid) → Learn (video/text)
  → Complete chapters → Progress tracked → 100% → Certificate generated
  → Join community → Chat → Submit projects → Get feedback

Instructor Journey:
  Register → Create course → Add sections/chapters → Upload videos (Supabase)
  → Publish → Students enroll → View analytics/earnings → Review submissions
  → Give project feedback → Host live sessions

Content Creation:
  Write article (EditorJS) → Save draft / Publish → Community reads/likes/comments
  Generate AI roadmap → View/download → Share


9. FRONTEND ARCHITECTURE
------------------------

App Router structure (src/app/):
  (auth)/         — login, register, forgot-password, reset-password, verify-email
  (public)/       — browse, courses, articles, projects, roadmaps, communityChat,
                    instructors, subscriptions, legal, feedback, feature-request
  (user)/         — dashboard, cart, wishlist, learnings, profile, settings, writeArticle
  (instructor)/   — courses, analytics, earnings, projects, sessions, students, settings
  (sessions)/     — browseSessions, bookSession, pastSessions, upcomingSessions
  api/auth/       — NextAuth route ([...nextauth])

State management:
  - Zustand stores: userStore, communityChatStore, cartStore, currencyStore,
    coursesStore, categoriesStore, articlesStore, notificationsStore,
    preferencesStore, learningGoalsStore, notificationPreferencesStore
  - All use SSR-safe storage (safeLocalStorage utility)
  - No mega store — each concern has its own store

API layer:
  - ApiManager singleton (axios-based) with:
    - Request interceptor: attach JWT token
    - Response interceptor: auto-refresh on 401
    - Rate limiting (client-side)
    - Dev-only logging (gated behind NODE_ENV)
  - Service classes: courseService, projectsService, authService, etc.
  - React Query for server state (5min stale, 10min cache)


10. DEPLOYMENT & ENVIRONMENT
-----------------------------

Environment variables required:

Frontend (.env):
  NEXT_PUBLIC_API_URL        — Backend API base URL
  NEXT_PUBLIC_APP_URL        — Frontend URL (for metadata)
  NEXTAUTH_SECRET            — NextAuth session secret
  NEXTAUTH_URL               — NextAuth callback URL
  GOOGLE_CLIENT_ID           — Google OAuth client ID
  GOOGLE_CLIENT_SECRET       — Google OAuth secret
  GITHUB_CLIENT_ID           — GitHub OAuth client ID
  GITHUB_CLIENT_SECRET       — GitHub OAuth secret

Backend (.env):
  DATABASE_URL               — PostgreSQL connection string
  JWT_SECRET                 — Access token secret (min 20 chars)
  JWT_REFRESH_SECRET         — Refresh token secret (min 20 chars)
  STRIPE_SECRET_KEY          — Stripe secret key
  STRIPE_WEBHOOK_SECRET      — Stripe webhook signing secret
  SENDGRID_API_KEY           — SendGrid API key
  FRONTEND_URL_PROD          — Frontend URL for CORS
  GOOGLE_CLIENT_ID           — (optional, for future backend OAuth validation)
  GITHUB_CLIENT_ID           — (optional, for future backend OAuth validation)
  SUPABASE_URL               — Supabase project URL
  SUPABASE_SERVICE_KEY       — Supabase service role key
